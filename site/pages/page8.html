8. Функции в ES6

Стрелочные функции

Стрелочные функции были введены в ES2015.

Если раньше единственной правильной записью функции была такая:

function sayMyName (name) {

console.log('My name is ' + name);

}

То теперь возможен и другой вариант. С помощью Function Expression мы также можем объявить стрелочную функцию — Arrow Function.

const sayMyName = name => {

console.log('My name is ' + name);

}

Так что нет необходимости записывать ключевое слово function.

Если параметр один, фигурные скобки {} не нужны:

const logText = text => console.log(text);

const sayHelloWorld = () => console.log('Hello, world!');

const sum = (a, b) => a + b;

Но если параметр один и с ним будут происходить какие-то манипуляции, тогда фигурные скобки так же будут нужны. Например:

const logUppercaseText = text => {

const uppercaseText = text.toUpperCase();

console.log(uppercaseText);

};

Если параметров не ожидается или же, наоборот, их больше одного, но с ним будут происходить какие-то манипуляции или в теле будет какая-то логика, фигурные скобки обязательны.

const fatBodyArrowFunction = (a, b, c) => {

// тут производятся сложные вычисления на несколько строк кода.

// если нужно что-то вернуть из такой функции — используйте return

// иначе функция будет вести себя как процедура

const result = a + b + c;

return result;

};

Особенности:

  • Короткий синтаксис.
  • Отсутствие привязки к this (подробнее поговорим об этом дальше).
  • Нельзя использовать как конструктор (подробнее поговорим об этом дальше).
  • Нет псевдомассива arguments.

Стрелочные функции не могут обратиться к параметру arguments во время вызова. Если попытаться это сделать, то произойдёт ошибка:

const func = () => console.log(arguments);

func() // Uncaught ReferenceError: arguments is not defined
